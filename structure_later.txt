Data Flow
----------

Frontend (React + Web3)
 └─ ComplaintForm
      └─ Upload image → IPFS/Pinata → get CID
      └─ Generate SHA-256 hash of complaintId + description + location + imageCID
      └─ Send hash & complaintId to Smart Contract
      └─ Optional: Send complaint metadata to Backend (MongoDB) for fast querying

Smart Contract (Polygon Mumbai)
 └─ storeHash(hash, complaintId, reporterWallet)
 └─ updateStatus(complaintId, status) [admin]
 └─ verifyHash(complaintId) → confirms tamper-proof

Backend (Node.js + MongoDB) [Optional]
 └─ Stores complaint data (description, location, imageCID, hash, status)
 └─ Provides REST API for frontend to fetch & display complaints

IPFS / Pinata
 └─ Stores uploaded images/videos
 └─ Returns CID → stored on-chain & in backend
